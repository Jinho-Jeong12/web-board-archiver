  1. 제작 목적
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  이 도구는 웹 게시판의 게시물을 증거 채증 목적으로 로컬에 보관하기 위해
  제작되었습니다.

  인터넷 게시물은 작성자가 삭제하거나 사이트가 폐쇄되면 영구적으로 사라집니다.
  이 도구를 이용해 게시물의 HTML 원본을 통째로 저장해두면,
  추후 법적 절차나 분쟁 시 해당 게시물의 존재 및 내용을 입증하는 자료로
  활용할 수 있습니다.

  ※ 수집한 자료는 적법한 목적 내에서만 사용하시기 바랍니다.

  2. GitHub에서 다운로드하는 방법
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
0. Python 먼저 설치해주세요

 https://www.python.org/downloads/

  1. 아래 주소로 접속합니다.
       https://github.com/Jinho-Jeong12/web-board-archiver

  2. 초록색 [ <> Code ▼ ] 버튼을 클릭합니다.

  3. 드롭다운 메뉴 하단의 [ Download ZIP ] 을 클릭합니다.

  4. ZIP 파일이 다운로드됩니다. 원하는 폴더에 압축을 풀어주세요.

  5. 압축 폴더 안에 web_board_archiver.py 와 README.txt 가 있습니다.

  ※ git이 설치되어 있다면 아래 명령어로 바로 받을 수도 있습니다:
       git clone https://github.com/Jinho-Jeong12/web-board-archiver.git

  3. 파일 안내
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  web_board_archiver.py  — 메인 스크립트 (이것만 사용하면 됩니다)

  ※ final_v15 / final_v17 파일은 구버전입니다.
     두 버전의 유일한 차이는 동시 작업자(워커) 수였으나,
     현재 버전은 속도 레벨(1~10)로 수집 속도를 직접 조절할 수 있어
     별도 버전을 유지할 필요가 없어졌습니다.

  4. 사전 준비 — Python 설치
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  이 스크립트는 Python 3.8 이상에서 동작합니다.

  [Windows — 설치 방법]
  1. https://www.python.org/downloads/ 에 접속합니다.
  2. "Download Python 3.x.x" 버튼을 클릭합니다.
  3. 설치 프로그램을 실행합니다.
     ★ 중요: 설치 화면 하단에 "Add Python to PATH" 체크박스가 있습니다.
             반드시 체크한 뒤 Install Now를 클릭하세요!
             체크하지 않으면 스크립트가 실행되지 않습니다.
  4. 설치가 완료되면 이 스크립트를 실행하면 됩니다.

  [macOS — 실행 방법]
  macOS에서는 .py 파일을 더블클릭하면 텍스트 에디터로 열립니다.
  아래 방법으로 실행하세요.

  1. web_board_archiver.py 파일을 우클릭합니다.
  2. "다음으로 열기" → "Python Launcher" 또는 "IDLE" 을 선택합니다.
  3. IDLE이 열리면 상단 메뉴에서 Run → Run Module 을 클릭합니다.
  4. 스크립트가 실행됩니다.

  ※ Python이 설치되어 있지 않다면 https://www.python.org/downloads/macos/
    에서 먼저 설치하세요.

  [패키지 자동 설치]
  스크립트를 처음 실행하면 필요한 패키지(requests, beautifulsoup4, lxml)를
  자동으로 설치합니다. 별도로 설치할 필요 없습니다.

  5. 실행 방법 (Windows)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [기본 실행]

    python web_board_archiver.py <게시판_URL>

  [URL 형식 예시]

    ■ DCInside 마이너 갤러리 (주소에 mgallery 포함):
      python web_board_archiver.py "https://gall.dcinside.com/mgallery/board/lists/?id=galleryname"

    ■ DCInside 일반 갤러리:
      python web_board_archiver.py "https://gall.dcinside.com/board/lists/?id=galleryname"

    ■ 다른 사이트 (link-pattern 필요):
      python web_board_archiver.py "https://example.com/board/" --link-pattern "/board/view/\?no=\d+"

  [URL 없이 실행하기]
    URL 없이 실행하면 프로그램이 직접 URL 입력을 요청합니다:

      python web_board_archiver.py

  6. 실행 방법 (macOS)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  macOS에서는 .py 파일을 더블클릭하면 텍스트 에디터로 열립니다.
  아래 방법으로 실행하세요.

  [방법 1 — IDLE로 실행 (권장)]

  1. web_board_archiver.py 파일을 우클릭합니다.
  2. "다음으로 열기" → "IDLE" 을 선택합니다.
  3. IDLE 창이 열리면 상단 메뉴에서 Run → Run Module 을 클릭합니다.
     (단축키: F5)
  4. URL 입력 창이 뜨면 수집할 게시판 URL을 붙여넣고 Enter.

  [방법 2 — URL을 미리 넣어서 실행하기]

  IDLE 하단 입력창(Shell)에 직접 명령어를 실행하거나,
  터미널(Terminal)에서 아래처럼 입력해도 됩니다:

    python3 web_board_archiver.py "https://gall.dcinside.com/mgallery/board/lists/?id=galleryname"

  [터미널 여는 방법]
    Finder → 응용 프로그램 → 유틸리티 → 터미널

  7. 실행 시 대화 순서
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  스크립트를 실행하면 아래 순서로 질문합니다.

  1. 사용 설명서 보기 여부 (y/n)
     y를 입력하면 프로그램 내장 설명서를 출력합니다.

  2. 요청 제한 사이트 여부 (y/n)
     수집 대상 사이트가 짧은 시간에 요청이 많으면 자동으로 접근을 막는
     사이트인지 선택합니다.

       y — 요청 제한 사이트: 요청 간격을 두고 안전하게 수집.
                             속도 레벨 5 이하 강력 권장.
       n — 일반 사이트:      제한 없이 빠르게 수집.
                             단, 실제로 제한에 걸리면 약 30~40분간
                             해당 사이트에 일시적으로 접근이 차단됩니다.

  3. 속도 레벨 선택 (1~10)
     아래 표를 참고해 원하는 레벨을 입력합니다.
     Enter만 누르면 권장 기본값이 자동 선택됩니다.
       요청 제한 사이트 → 기본값 5
       일반 사이트      → 기본값 8

  8. 속도 레벨 상세 설명 (1~10)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  레벨  요청 간격       초당 처리량   일시 차단 위험          비고
  ────  ────────────    ──────────    ────────────────────    ──────────────────
   1    4.0 ~ 6.0초     약 0.2개      거의 없음               가장 안전
   2    3.0 ~ 4.5초     약 0.3개      낮음
   3    2.0 ~ 3.0초     약 0.4개      낮음
   4    1.5 ~ 2.5초     약 0.5개      보통
   5    0.9 ~ 1.8초     약 0.8개      보통                    ← 요청 제한 사이트 기본 권장
   6    0.5 ~ 1.0초     약 1.3개      약간 높음
   7    0.3 ~ 0.6초     약 2.2개      높음
   8    0.1 ~ 0.3초     약 5개        매우 높음
   9    0.05 ~ 0.1초    약 15개       일시 차단 시 약 30~40분 접근 불가
  10    0.0 ~ 0.05초    약 30개+      요청 제한 사이트에서 수분 내 일시 차단 확실

  ※ "일시 차단"이란 해당 사이트가 요청이 너무 많다고 판단해
     내 IP를 약 30~40분간 자동으로 막는 것을 의미합니다.
     차단이 풀린 뒤 다시 실행하면 이어받기가 가능합니다.

  9. 저장 파일명 규칙
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  게시물은 "게시물번호_제목.html" 형태로 저장됩니다.

  예시:
    123456_오늘 점심 뭐먹지.html
    123457_주말 여행 후기.html
    123458_질문 있어요.html

  제목을 가져올 수 없는 경우(삭제된 게시물 등):
    123459.html  ← 번호만 사용

  ★ 파일명 앞의 번호 덕분에 숫자 순으로 정렬되어 게시물 순서가 유지됩니다.
  ★ 파일명에 사용할 수 없는 특수문자(\ / : * ? " < > |)는 자동으로 제거됩니다.
  ★ 제목이 80자를 초과하면 자동으로 잘립니다.

  10. 저장 위치
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  수집된 게시물은 HTML 파일로 저장됩니다.

    DCInside:    스크립트 폴더\posts\dcinside\<갤러리ID>\<번호_제목>.html
    기타 사이트: 스크립트 폴더\posts\<사이트명>\<번호_제목>.html

  예시:
    posts\dcinside\mygallery\123456_오늘 점심 뭐먹지.html
    posts\dcinside\mygallery\123457_주말 여행 후기.html
    ...

  --save-dir 옵션으로 저장 위치를 원하는 폴더로 바꿀 수 있습니다:
    python web_board_archiver.py "URL" --save-dir "D:\my_archive"


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  11. 자동 생성 파일 설명
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  스크립트와 같은 폴더에 아래 파일들이 자동 생성됩니다.

  archiver_ids_cache.txt
    수집 대상 게시물 번호 목록을 저장하는 캐시 파일입니다.
    한 번 목록을 수집하면 이 파일에 저장해두고, 재실행 시에는
    다시 목록을 긁지 않고 이 파일을 바로 읽어 시간을 절약합니다.
    ★ 다른 갤러리로 바꿀 때는 반드시 이 파일을 삭제하세요!
       삭제하지 않으면 이전 갤러리 목록을 그대로 사용합니다.

  archiver_progress.txt
    성공적으로 저장된 게시물 번호 목록입니다.
    재실행 시 이미 완료된 게시물은 건너뜁니다.

  archiver_fail.txt
    저장에 실패한 게시물 번호 목록입니다.
    재실행 시 실패한 게시물을 자동으로 다시 시도합니다.
    최종적으로 계속 실패하는 경우 삭제된 게시물일 가능성이 높습니다.

  archiver.log
    수집 중 발생한 모든 이벤트의 로그 파일입니다.
    오류가 발생하면 이 파일을 확인하세요.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  12. 이어받기 / 재실행
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  중단된 작업을 이어받으려면 그냥 같은 명령어로 재실행하면 됩니다.
  - 이미 저장된 파일은 다시 내려받지 않습니다.
  - 이전 실패 파일은 자동 재시도합니다.
  - 목록 캐시가 있으면 목록 재수집을 건너뜁니다.

  처음부터 완전히 다시 시작하려면:
    archiver_ids_cache.txt, archiver_progress.txt, archiver_fail.txt 삭제 후 재실행


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  13. 옵션 전체 목록
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  --link-pattern REGEX
      게시물 링크를 골라낼 정규식 패턴입니다.
      DCInside 외 사이트에서 게시물 링크만 수집하고 싶을 때 사용합니다.

      예시: 아래 형태의 URL만 게시물로 인식하고 싶을 때
            https://example.com/board/view/?no=123
        → --link-pattern "/board/view/\?no=\d+"

  --page-param NAME
      게시판 페이지를 넘길 때 사용하는 쿼리 파라미터 이름입니다.
      기본값: page

      예시: URL이 ?p=2, ?p=3 형태인 사이트
        → --page-param p

  --save-dir PATH
      HTML 파일 저장 폴더를 직접 지정합니다.
      기본값: 스크립트 위치\posts\<사이트>\<갤러리ID>\

      예시:
        → --save-dir "D:\archive\my_gallery"

  14. 일시 차단 발생 시 대처법
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  수집 중 429 또는 403 오류가 반복되면 일시 차단된 것입니다.

  1. 스크립트를 종료합니다 (Ctrl+C).
  2. 약 30~40분 기다립니다.
  3. 같은 명령어로 재실행합니다. (이어받기 자동 적용)
  4. 속도 레벨을 더 낮게 설정하세요.

  archiver_fail.txt 에 실패한 게시물 번호가 저장되어 있으므로
  재실행 시 실패한 것들을 자동으로 다시 시도합니다.

  Q. 다른 컴퓨터에서도 쓸 수 있나요?
  A. Python만 설치되어 있으면 됩니다. 패키지는 첫 실행 시 자동 설치됩니다.
     archiver_ids_cache.txt 파일을 함께 복사하면 목록 재수집도 건너뜁니다.

  Q. macOS에서 파일을 더블클릭했는데 텍스트 에디터로 열려요.
  A. .py 파일을 우클릭 → "다음으로 열기" → IDLE 선택 후,
     상단 메뉴 Run → Run Module 을 클릭하면 실행됩니다.

  Q. 갤러리를 바꾸려면 어떻게 하나요?
  A. URL을 바꾸고, archiver_ids_cache.txt 파일을 삭제한 뒤 재실행하세요.
     archiver_progress.txt와 archiver_fail.txt도 삭제해야 새 갤러리 기준으로 작동합니다.

  Q. 수집 중간에 컴퓨터를 껐다가 켜도 되나요?
  A. 됩니다. 이미 저장된 파일은 다시 내려받지 않고 중단 지점부터 이어받습니다.

  Q. 파일이 "번호_제목" 형태로 저장되는데, 검색은 어떻게 하나요?
  A. 파일명 자체가 제목이므로 탐색기에서 제목 키워드로 바로 검색됩니다.
     파일명 앞의 번호 순으로 정렬하면 게시물 올라온 순서를 유지합니다.

  Q. 게시물 제목이 파일명에 제대로 안 나와요.
  A. 제목을 가져올 수 없는 경우(삭제된 게시물, 로그인 필요 게시물 등)
     번호만 파일명으로 사용됩니다. (예: 123456.html)

  Q. v15, v17 파일은 어떻게 하나요?
  A. 구버전입니다. 삭제해도 무방합니다.
     현재 버전(web_board_archiver.py)에서 속도 레벨로 동일하게 조절 가능합니다.

  Q. 게시물이 저장됐는데 내용이 비어있거나 이상해요.
  A. 해당 게시물이 삭제되었거나 로그인 없이는 볼 수 없는 경우입니다.

  Q. 오류 내용을 확인하고 싶어요.
  A. archiver.log 파일을 메모장으로 열면 전체 로그가 기록되어 있습니다.
